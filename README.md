## 今回から新しく実装した関数
1. Or-opt  
   連続した1〜3都市の塊を一旦抜いて、それを別の場所に入れ直し、距離がよくなら採用する。というアルゴリズム  
   長さは3から順番に、場合によっては1まで試す。  
   route上の連続区間（1〜3都市）を1つ選び、その区間を一旦取り除く。  
   その後、残ったルートの全ての辺に対して挿入コストを評価し、最も改善する位置に挿入する。  
   改善が見つかった場合のみルートを更新し、ループを継続する。  

3. ダブルブリッジ法  
   局所最適を強制的に壊して、新しい探索領域へ飛ばす操作。   
   ランダムに3つの分割点を選び、それによって巡回路を4つの連続区間に分割する。   
   その後、それらのブロックの順序を入れ替えて新しい巡回路を生成することで、局所最適からの脱出を図る。    

4. 領域結合と領域分割法  
   大きい領域の場合に、それを分割して別々に解き、それを後から合わせるという関数  
   都市数が150以下の場合は分割を行わない。  
   分割を行う場合は、x座標とy座標の分布を見て、縦と横どちらに分割するのかを決める。  
   半分に分割したら、自分自身を呼ぶことで、再帰的に分割を行う。  
   分割は都市数が150以下になるまで繰り返し、その後は貪欲法で初期解を作り、2-optで局所改善を行う。  
   最後に、部分解同士をmerge_toursで最適に接続する。  
   merge_toursでは、２つの巡回路を、1つの巡回路にする最適な繋ぎ方を探す。  
   全ての辺の組み合わせを試し、最も良いものを使う。  

6. main関数の変更  
   初期解を作る -> 局所改善 -> 焼きなまし -> ダブルブリッジ法(5回) -> 出力  
   初期解を作る過程で、領域分割や貪欲法、2-OPTを行っている。
